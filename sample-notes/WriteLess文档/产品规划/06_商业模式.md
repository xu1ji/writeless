```
┌─────────────────────────────────────────────────────────────────────────┐
│                     WriteLess 单元模型                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │

│   │        │        │        │        │                                │
```

## 收入模式

### 定价策略

| 方案 | 价格 | 目标用户 | 说明 |
|------|------|----------|------|
| **免费版** | $0 | 轻度用户 | 基础功能，AI 有配额限制 |
| **Pro 版** | $9.9/月 | 个人用户 | 完整功能，AI 无限制 |
| **Team 版** | $19.9/用户/月 | 团队用户 | 多人协作，管理后台 |

**定价逻辑**：
- 免费版：获客 + 留存工具
- Pro 版：核心收入来源
- Team 版：高 ARPU 用户

### 收入预测

| 指标 | 保守 | 中等 | 乐观 |
|------|------|------|------|
| 用户数（1 年） | 1000 | 5000 | 20000 |
| 付费率 | 5% | 10% | 15% |
| 付费用户 | 50 | 500 | 3000 |
| ARPU（月） | $10 | $10 | $12 |
| 月收入 | $500 | $5000 | $36000 |
| 年收入 | $6K | $60K | $432K |

## 单元模型验证

### 核心假设

> **假设**：单个用户的 LTV > 3×CAC，单元模型能盈利。

### 关键指标

| 指标 | 假设值 | 验证方式 |
|------|--------|----------|
| **CAC** | $3-5 | 追踪获客渠道 |
| **LTV** | $50-100 | 用户付费周期 × ARPU |
| **LTV/CAC** | 10-20x | 计算 |
| **毛利率** | 80%+ | (收入 - 变动成本) / 收入 |
| **回本周期** | < 3 个月 | CAC / 月 ARPU |

### 盈利信心评估

| 维度 | 评估 | 说明 |
|------|------|------|
| **CAC 可控吗？** | ✅ 是 | 开源项目，自然增长为主 |
| **LTV 可预期吗？** | ⚠️ 待验证 | 需验证付费率和留存率 |
| **毛利率健康吗？** | ✅ 是 | 变动成本低 |
| **回本快吗？** | ✅ 是 | 预计 < 3 个月 |

## 盈利路径

### 阶段 1：验证期（0-1000 用户）

```
目标：验证付费意愿

关键指标：
├── 付费率 > 5%
├── 月留存率 > 30%
└── NPS > 40

不做：
├── 不投付费广告
├── 不做大规模推广
└── 不急于盈利
```

### 阶段 2：增长期（1000-10000 用户）

```
目标：验证规模化盈利

关键指标：
├── 付费用户 > 500
├── 月收入 > $5000
└── 单元模型健康

开始：
├── 内容营销
├── 社区运营
└── 口碑激励
```

### 阶段 3：规模化（10000+ 用户）

```
目标：规模化盈利

关键指标：
├── 年收入 > $100K
├── 团队版占比 > 20%
└── 可以雇佣 1-2 人
```

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| **付费率低于 5%** | 中 | 高 | 调整定价或增加免费版价值 |
| **AI 成本过高** | 低 | 中 | 优化本地模型使用 |
| **竞品降价** | 中 | 中 | 专注差异化，不打价格战 |
| **用户留存差** | 中 | 高 | 优化产品内核，提升粘性 |
