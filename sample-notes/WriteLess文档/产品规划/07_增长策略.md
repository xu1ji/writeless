# 07. 增长策略

> 未来用什么方法获客？

---

## 核心问题

WriteLess 如何以可控成本获取用户？增长渠道和毛利是否匹配？

---

## 渠道选择原则

```
增长渠道选择原则：
├── 渠道要和毛利匹配（高毛利能投广告，低毛利靠自然增长）
└── 渠道要和人群匹配（用户在哪，就去哪）
```

---

## 毛利分析

### 预估毛利率

| 成本项 | Pro 版 ($9.9/月) | Team 版 ($19.9/月) |
|--------|------------------|-------------------|
| AI API 成本 | $1-3 | $2-5 |
| 托管成本 | $0.5 | $1 |
| 支付手续费 | $0.3 | $0.6 |
| **总成本** | $1.8-3.8 | $3.6-6.6 |
| **毛利率** | 62-82% | 67-82% |

**结论**：毛利率约 60-80%，属于**中等毛利**产品

### 渠道匹配矩阵

```
                    毛利高
                       │
     付费广告          │        销售团队
     SEM/信息流        │        大客户销售
        ❌            │           ❌
                       │
    ───────────────────┼────────────────────
      用户少           │            用户多
                       │
     内容营销          │        口碑传播
     GitHub/社区       │        开源社区
        ✅            │           ✅
                       │
                    毛利低（WriteLess 约 60-80%）
```

## 增长渠道规划

### 渠道 1：GitHub 开源（主渠道）

**为什么选择**：
- 零成本获客
- 技术人群精准匹配
- 开源社区自带传播

**执行策略**：

| 阶段 | 动作 | 目标 |
|------|------|------|
| **冷启动** | 开源核心功能，完善 README | 100+ Stars |
| **成长期** | 加入 GitHub Trending | 500+ Stars |
| **成熟期** | 建立插件生态 | 1000+ Stars |

**内容要点**：
- 清晰的 README（问题 → 解决方案 → 安装 → 使用）
- 演示视频（30 秒内展示核心价值）
- 完善的文档
- 贡献指南

**预估 CAC**：$0-2

---

### 渠道 2：Product Hunt 发布

**为什么选择**：
- 一次发布，长期流量
- 产品人群精准
- 背书效应

**执行策略**：

| 动作 | 说明 |
|------|------|
| **准备期** | 完善 Product Hunt 页面，准备演示视频 |
| **发布日** | 邀请早期用户支持，争取 Top 5 |
| **后续** | 利用 PH 徽章增加信任 |

**时间**：产品 MVP 完成后发布

**预估 CAC**：$1-5

---

### 渠道 3：内容营销

**为什么选择**：
- 长期积累，复利效应
- 建立专业形象
- 精准触达目标用户

**内容方向**：

| 内容类型 | 平台 | 频率 |
|----------|------|------|
| **技术文章** | 掘金、知乎、Medium | 2 周 1 篇 |
| **视频教程** | B站、YouTube | 1 月 1 个 |
| **Newsletter** | 个人博客/公众号 | 2 周 1 期 |

**内容主题**：
- "为什么我放弃了 Obsidian"
- "AI 原生笔记的工作流"
- "本地优先 vs 云端笔记"
- "分层索引如何比 RAG 更精准"

**预估 CAC**：$3-10

---

### 渠道 4：社区运营

**为什么选择**：
- 口碑传播成本低
- 早期用户是最好的推广者
- 反馈直接

**执行策略**：

| 动作 | 说明 |
|------|------|
| **Discord/微信群** | 建立用户社区 |
| **用户案例** | 收集并分享用户故事 |
| **定期 Office Hour** | 和用户直接交流 |

**预估 CAC**：$0-3

---

### 渠道 5：口碑传播

**为什么选择**：
- 成本最低
- 转化率最高
- 信任度最强

**激励机制**：

| 机制 | 说明 |
|------|------|
| **免费 Pro 月** | 邀请好友送 1 个月 Pro |
| **功能解锁** | 邀请 3 人解锁高级功能 |
| **社区荣誉** | 贡献者徽章 |

**预估 CAC**：$0

---

## 不做的渠道

| 渠道 | 为什么不做 |
|------|-----------|
| **付费广告** | 毛利不够高，CAC 难以控制 |
| **信息流投放** | 目标用户不是大众用户 |
| **销售团队** | 客单价低，不值得 |
| **KOL 合作** | 早期预算有限 |

## 分阶段增长计划

### 阶段 1：验证期（0-1000 用户）

```
目标：验证产品内核

渠道：
├── GitHub 开源（主）
├── 个人网络
└── 少数派/利器等小众媒体

不做：
├── 不投付费广告
├── 不做大规模推广
└── 不急于用户增长

关键指标：
├── 周留存率 > 30%
├── 付费率 > 5%
└── NPS > 40
```

### 阶段 2：增长期（1000-10000 用户）

```
目标：规模化验证

渠道：
├── Product Hunt 发布
├── 内容营销（持续）
├── 社区运营
└── 口碑激励

开始投入：
├── 每周 1 篇内容
├── 社区定期活动
└── 用户案例收集

关键指标：
├── 月增长 > 20%
├── 自然流量占比 > 60%
└── 付费用户 > 500
```

### 阶段 3：规模化（10000+ 用户）

```
目标：规模化增长

渠道：
├── 全渠道运营
├── 插件生态
├── 企业版销售
└── 国际化

关键指标：
├── 年收入 > $100K
├── 企业客户 > 50
└── 团队版占比 > 20%
```

## 验证方法

### 核心假设

> **假设**：通过 [渠道]，能以 [成本] 获取 [数量] 用户

### 验证步骤

1. **小规模测试**：每个渠道投入最小资源
2. **计算 CAC**：花费 / 获得用户数
3. **评估质量**：留存率、付费率
4. **决定是否放大**

### 成功标准

- [ ] 找到 1-2 个 CAC < $5 的渠道
- [ ] 自然流量占比 > 50%
- [ ] 渠道用户留存率 > 30%
