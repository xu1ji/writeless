# WriteLess 前端设计规范

> 用于 AI 生成前端代码的设计指南

---

## 整体设计风格

### 核心原则
- **极简主义**：去除一切不必要的装饰
- **内容优先**：让笔记内容成为视觉焦点
- **平滑过渡**：编辑态和预览态无缝切换
- **无干扰**：沉浸式写作体验

### 设计参考
- **Obsidian** 的 Live Preview 交互模式
- **Notion** 的极简白色美学
- **Typora** 的所见即所得体验

### 主色调
```css
/* 背景色系 */
--bg-primary: #ffffff;        /* 主背景 - 纯白 */

/* 文字色系 */

/* 边框色系 */

## 组件设计

### 1. 文件树节点

```
┌─────────────────────────────┐
│ ▼ 📁 文件夹名称              │  ← 展开/折叠箭头 + 图标 + 名称
│   └─ 📄 子文件.md           │  ← 缩进 + 文件图标 + 名称
│   └─ 📄 另一个文件.md        │
│   └─ ▶ 📁 子文件夹           │  ← 可嵌套
└─────────────────────────────┘
```

**交互**：
- 悬停：背景变为 `var(--bg-tertiary)`
- 点击文件：打开编辑
- 点击文件夹：展开/折叠
- 选中：背景 `var(--bg-tertiary)` + 左边框 2px 蓝色

### 2. Markdown 编辑器（核心）

**Obsidian Live Preview 模式**：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  # 这是一级标题                              ← 渲染显示          │
│                                                                 │
│  这是正文内容，**粗体文字** 会被加粗显示                         │
│  而不是显示星号。                                               │
│                                                                 │
│  - 列表项会显示为圆点             ← 光标不在时渲染              │
│  - 第二项                        ← 点击后显示 - 语法            │
│  - [ ] 待办事项                  ← 显示为复选框                 │
│                                                                 │
│  > 引用块会有左边框和灰色背景                                    │
│                                                                 │
│  `行内代码` 会显示为灰色背景                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

当光标进入某一行时：
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  # 这是一级标题                                                  │
│                                                                 │
│  这是正文内容，**粗体文字|** 会被加粗   ← 光标处显示 ** 语法     │
│  │                     ↑                                        │
│  └─ 光标在这里时，** 显示出来                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键交互**：
1. **非编辑态**：Markdown 语法隐藏，显示渲染结果
2. **编辑态**：光标所在行/块显示原始 Markdown 语法
3. **平滑过渡**：语法显示/隐藏有动画过渡
4. **块级编辑**：每个段落/标题/列表是独立的可编辑块

### 3. 待办事项

```
┌─────────────────────────────────────────────────────────────────┐
│ ☑ 已完成的任务                    ← 划线 + 灰色                 │
│ ☐ 未完成的任务                    ← 可点击切换                   │
│ ☐ 另一个未完成任务                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 代码块

```
┌─────────────────────────────────────────────────────────────────┐
│ ```javascript                          ┌────────┐               │
│ function hello() {                     │ Copy ▼ │ ← 操作按钮    │
│   console.log('Hello World');          └────────┘               │
│ }                                                               │
│ ```                                                             │
└─────────────────────────────────────────────────────────────────┘
```

## 动效规范

### 过渡动画
```css
--transition-fast: 0.1s ease;      /* 悬停等快速反馈 */
--transition-base: 0.2s ease;      /* 常规过渡 */
--transition-slow: 0.3s ease;      /* 展开/折叠 */
```

### 常用动画
```css
/* 悬停背景 */
.hover-bg {
  transition: background-color var(--transition-fast);
}

/* 展开/折叠 */
.collapse {
  transition: height var(--transition-slow),
              opacity var(--transition-slow);
}

/* Markdown 语法显隐 */
.syntax-toggle {
  transition: opacity var(--transition-base);
}
```

## 响应式断点

```css
/* 暂只考虑桌面端 */
--breakpoint-sm: 640px;   /* 小屏 */
--breakpoint-md: 768px;   /* 平板 */
--breakpoint-lg: 1024px;  /* 桌面 */
--breakpoint-xl: 1280px;  /* 大屏 */
```

### 断点行为
| 断点 | 侧边栏 | 内容宽度 |
|------|--------|----------|
| < 768px | 隐藏（汉堡菜单） | 100% |
| 768px - 1024px | 200px | 640px |
| > 1024px | 240px | 720px |
