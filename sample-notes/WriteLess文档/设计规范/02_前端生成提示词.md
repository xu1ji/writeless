# WriteLess 前端生成提示词

> 直接复制此提示词给 AI，生成 WriteLess 前端界面

---

## 完整提示词

```
请帮我创建一个 Markdown 笔记应用的前端界面，名为 WriteLess。

## 一、整体风格

参考 Obsidian 和 Notion 的设计：
- 极简白色背景，干净无干扰
- 暖灰白色调（#f7f6f3）用于侧边栏
- 主文字颜色 #37352f（Notion 标准）
- 内容居中，最大宽度 720px
- 无阴影、无渐变、纯扁平设计

## 二、布局结构

采用左右分栏布局：

### 左侧侧边栏（240px）
- 背景色 #f7f6f3
- 顶部有"打开文件夹"按钮
- 下方显示文件树结构
- 文件夹可展开/折叠（▶/▼ 箭头）
- 文件图标 + 文件名
- 悬停时背景变为 #eceae5
- 选中时左边框 2px 蓝色高亮

### 右侧编辑区（flex-1）
- 白色背景
- 内容区居中，max-width: 720px
- 上下边距 32px，左右边距 48px
- 滚动时整个区域滚动，不是内容区滚动

## 三、核心功能：Markdown Live Preview 编辑器

这是最重要的部分，参考 Obsidian 的 Live Preview 模式：

### 编辑体验
1. **块级编辑**：每个段落/标题/列表是独立的可编辑块
2. **语法隐藏**：
   - 光标不在时：Markdown 语法隐藏，显示渲染结果
   - 光标进入时：显示原始 Markdown 语法
   - 例如：**粗体** 正常显示为粗体文字，光标进入时显示 **粗体**
3. **平滑过渡**：语法显示/隐藏有 0.2s 的过渡动画
4. **实时渲染**：无需切换预览，所见即所得

### 待办事项
- `- [ ] 任务` 显示为可点击的复选框 ☐
- `- [x] 任务` 显示为已勾选 ☑ + 划线 + 灰色文字
- 点击复选框切换状态

### 其他元素渲染
- 标题：H1 30px, H2 24px, H3 20px, 加粗
- 引用块：左边框 3px 灰色 + 左边距 + 斜体灰色文字
- 代码块：灰色背景 #f7f6f3 + 等宽字体 + 复制按钮
- 行内代码：浅灰背景 + 红色文字 #eb5757
- 链接：蓝色 #2383e2，悬停下划线
- 列表：实心圆点，嵌套缩进 16px

## 四、技术要求

- React 18 + TypeScript
- Tailwind CSS 样式
- CodeMirror 6 作为编辑器核心（必须）
- 使用 @codemirror/lang-markdown 解析
- Lucide React 图标库
- 响应式：桌面优先，侧边栏可收起

## 五、需要创建的组件

1. **App.tsx** - 主布局
2. **Sidebar.tsx** - 侧边栏
   - OpenFolderButton 组件
   - FileTree 组件（递归渲染）
   - FileNode 组件（单个文件/文件夹）
3. **Editor.tsx** - 编辑器
   - 集成 CodeMirror 6
   - 实现 Live Preview 效果
   - 自动保存（onChange debounce 500ms）
4. **文件类型定义** - types/index.ts

## 六、样式细节

### 字体
- 主字体：系统默认 (-apple-system, BlinkMacSystemFont...)
- 代码：SF Mono 或 JetBrains Mono

### 字号
- 侧边栏：14px
- 正文：16px
- H3：20px
- H2：24px
- H1：30px

### 间距
- 4px 基准网格
- 侧边栏内边距：12px
- 文件树每项高度：28px
- 每级缩进：16px

## 八、优先级

1. P0：侧边栏文件树 + 基础编辑器（可输入）
2. P0：Live Preview 效果（语法隐藏/显示）
3. P1：待办事项复选框交互
4. P2：自动保存
5. P3：文件树展开/折叠动画

请先创建项目基础结构，然后逐步实现各组件。
```
