# Prompt 工程最佳实践

## 1. 明确角色设定

```
你是一个专业的技术文档写手，擅长将复杂概念解释得通俗易懂。
```

## 2. 提供上下文

```
背景：我正在开发一个本地优先的笔记应用 WriteLess。
目标：实现类似 Obsidian 的 Live Preview 功能。
当前问题：光标位置检测不准确。
```

## 3. 结构化输出要求

```
请按以下格式输出：
1. 问题分析
2. 解决方案
3. 代码示例
4. 注意事项
```

## 4. Few-shot 示例

```
示例输入："实现一个计数器"
示例输出：
```typescript
const [count, setCount] = useState(0)
```

现在请实现：一个带最大值限制的计数器
```

## 5. 分步引导

对于复杂任务，分步引导：

```
第一步：分析需求
第二步：设计数据结构
第三步：实现核心逻辑
第四步：添加边界处理
```

## 6. 约束条件

```
要求：
- 代码使用 TypeScript
- 样式使用 Tailwind CSS
- 不要使用第三方库
- 代码要有详细注释
```

## 7. 迭代优化

```
上面的方案有个问题：性能不够好。
请优化：
1. 添加缓存机制
2. 使用虚拟滚动
3. 减少 re-render
```

## 常用 Prompt 模板

### 代码生成

```
任务：[描述功能]
技术栈：[React/TypeScript/Tailwind]
要求：[具体要求]
输出：完整代码 + 简要说明
```

### 代码审查

```
请审查以下代码：
[代码]

关注点：
1. 性能问题
2. 安全隐患
3. 代码风格
4. 可维护性
```

### 问题诊断

```
问题描述：[描述现象]
环境：[浏览器/Node 版本等]
已尝试：[已尝试的解决方案]
期望结果：[期望的行为]
```
