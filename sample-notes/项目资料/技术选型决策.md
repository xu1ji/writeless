# 技术选型决策记录

## 决策 1：选择 CodeMirror 6 而非 Slate.js

**日期**：2024-03-13

### 背景
需要实现 Live Preview 的 Markdown 编辑器。

### 候选方案
1. CodeMirror 6 - 专业代码编辑器
2. Slate.js - 富文本编辑框架
3. ProseMirror - 通用编辑器框架

### 决策
选择 **CodeMirror 6**

### 理由
1. **原生 Markdown 支持** - 有官方 @codemirror/lang-markdown
2. **装饰器系统完善** - 适合实现语法隐藏
3. **Obsidian 已验证** - 证明可行
4. **学习曲线较低** - 相比 Slate.js

### 风险
- 文档相对较少
- 部分边界情况需要自己处理

## 决策 2：选择 Tailwind CSS 4

**日期**：2024-03-13

### 背景
需要快速实现 Notion 风格的 UI。

### 候选方案
1. Tailwind CSS 4 - 原子化 CSS
2. CSS-in-JS（styled-components）
3. 传统 CSS + Sass

### 决策
选择 **Tailwind CSS 4**

### 理由
1. **开发速度快** - 不需要写 CSS 文件
2. **设计系统** - 内置 spacing、colors 体系
3. **体积小** - v4 使用 Oxide 引擎，更快
4. **与 React 配合好** - 类名直接写在 JSX

## 决策 3：桌面方案选择 Tauri

**日期**：2024-03-14

### 背景
需要将 Web 应用打包为桌面应用。

### 候选方案
1. Electron - 成熟但体积大
2. Tauri - 新兴但轻量

### 决策
选择 **Tauri**

### 理由
1. **体积小** - 10MB vs Electron 200MB+
2. **性能好** - 使用系统 WebView
3. **安全** - Rust 后端更安全
4. **差异化** - 可以"比 Obsidian 更轻"作为卖点

### 风险
- 需要学习基础 Rust
- 生态不如 Electron 成熟
